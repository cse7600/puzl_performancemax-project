**기능명:** 광고 소재 자동 판정 (Creative Performance Auto-Judge)
**버전:** 1.1
**작성일:** 2026-02-27
**최종 수정:** 2026-02-27 (ROAS 모드·계층구조·학습기간 추가)
**상태:** 기획 확정

---

## 1. 기능 개요

### 문제 정의
퍼포먼스 마케터는 매일 수십~수백 개의 광고 소재를 운영한다.
각 소재가 "계속 써야 하는지 / 중지해야 하는지 / 예산을 늘려야 하는지"를 판단하려면:
- CPA/ROAS, CTR, CPC, Frequency 등 다수 지표를 복합적으로 봐야 한다
- 캠페인 > 광고세트 > 광고 계층 구조를 이해하고 어느 레벨의 문제인지 파악해야 한다
- 업종 평균 대비 현재 성과가 어느 수준인지 알아야 한다
- 추세(최근 3일 vs 7일)까지 봐야 정확하다

이 과정을 수작업으로 하면 숙련된 시니어도 계정 1개에 30~60분 소요. **주니어는 판단 자체가 어렵다.**

### 솔루션
성과 데이터를 업로드하면 알고리즘이 각 소재를 **LIVE / OFF / WATCH / REVIVE / 학습 중 / 보류** 6단계로 자동 분류하고, "왜 이 판정이 나왔는지"와 "무엇을 해야 하는지"를 같이 알려준다.

## 3. 판정 모드

### 3.1 판정 모드 선택

사용자가 두 가지 판정 기준 중 하나를 선택한다.

| 모드 | 기준 지표 | 사용 대상 |
|------|----------|----------|
| **CPA 모드** | 전환당 비용 (원) | 리드, 설치, 회원가입 등 건당 단가가 중요한 경우 |
| **ROAS 모드** | 광고비 대비 매출 (%) | 이커머스, 직접 매출 추적이 가능한 경우 |

> 두 모드는 완전히 독립적으로 작동한다. 하나의 분석에서 동시에 두 모드를 사용하지 않는다.

### 3.2 ROAS 모드 판정 공식

```
ROAS = (전환 매출 합계 / 광고비) × 100%

예시:
  광고비 100,000원, 전환 매출 350,000원 → ROAS = 350%
```

| 상태 | ROAS 조건 | 권장 액션 |
|------|----------|-----------|
| **LIVE** | 목표 ROAS 이상 + 충분한 전환 | 유지 또는 예산 확대 |
| **LIVE★** | 전환 소수지만 ROAS 목표의 150% 이상 | 점진적 예산 증액 |
| **OFF** | ROAS가 목표의 50% 이하 (손실 구간) | 즉시 중지 |
| **WATCH** | ROAS 목표 50~80% 구간 + 추가 경고 | 소재 리프레시 검토 |
| **REVIVE** | ROAS 목표 80~100% 구간 | 설정 점검 후 재테스트 |
| **학습 중** | Meta 광고세트 학습 기간 중 | 판단 보류 — 학습 완료 후 재평가 |
| **보류** | 데이터 부족 | 데이터 더 쌓은 후 재평가 |

**ROAS 모드 추가 입력값:**

| 파라미터 | 설명 | 기본값 |
|---------|------|--------|
| 목표 ROAS | 손익분기 이상의 목표 광고 수익률 | 300% |
| OFF 배율 | 목표 ROAS의 N% 미만 시 OFF | 50% |

## 4. 판정 로직 (CPA 모드 기준 — 현재 v2.7 구현)

### 4.1 판정 상태 정의

| 상태 | 의미 | 권장 액션 |
|------|------|-----------|
| **LIVE** | 목표 CPA 이하, 충분한 전환 발생 | 유지 또는 예산 +10~20% |
| **LIVE★** | 전환 소수지만 CPA 목표의 50% 이하 (초효율) | 점진적 예산 증액 테스트 |
| **OFF** | CPA 기준 초과 또는 지속 전환 0건 | 즉시 중지 |
| **WATCH** | 경고 2개 이상 (CPA 경계구간 + CTR 하락 등) | 소재 리프레시 검토 |
| **REVIVE** | 경고 1개 또는 성과 미달 | 24~48h 재테스트 |
| **학습 중** | Meta 광고세트가 학습 기간 중 (`LEARNING` / `LEARNING_LIMITED`) | 판단 보류 |
| **보류** | 데이터 부족 (최소 지출 미달) | 데이터 더 쌓은 후 재평가 |

### 4.2 Meta 학습 기간 (Learning Phase) 처리

**API 호출 가능 여부:** Meta Marketing API의 AdSet 객체에서 `learning_phase_info` 필드로 직접 조회 가능.

```
AdSet.learning_phase_info.status 값:
  LEARNING           → 현재 학습 중 (최적화 데이터 수집 중)
  LEARNING_LIMITED   → 학습 제한됨 (전환 수 부족 등)
  NOT_LEARNING       → 학습 완료 또는 비활성
```

**판정 규칙:**
- API 연동 시: AdSet 상태가 `LEARNING` 또는 `LEARNING_LIMITED`이면 해당 광고세트의 모든 소재를 **"학습 중"** 으로 판정. CPA/ROAS 판정 우선순위보다 앞에 배치.
- CSV 업로드 시: 7일 내 전환 수가 10건 미만 + 지출이 목표 CPA × 3 미만인 경우 "학습 중(추정)" 배지 표시.
- 학습 중 소재는 **OFF 판정 불가** — 반드시 학습 완료 후 재판정.

> Meta 기준: Purchase 최적화 캠페인은 최소 10건(2025년 기준 하향됨), 일반 캠페인은 50건 이상이어야 학습 완료.

### 4.3 CPA 모드 판정 기준 입력값

| 파라미터 | 설명 | 기본값 |
|---------|------|--------|
| 목표 CPA | 전환 1건당 허용 비용 | 15,000원 |
| 최소 지출 기준 | 판정 가능한 최소 일일 지출 | 5,000원/일 |
| OFF 배율 (7일) | 목표 CPA × N배 초과 시 OFF | 2.0× |
| LIVE 배율 | 목표 CPA × N배 이하 시 LIVE | 1.2× |
| CTR 하락 한계 | CTR이 중앙값 대비 N% 이상 하락 시 경고 | -30% |
| CPC 상승 한계 | CPC가 중앙값 대비 N% 이상 상승 시 경고 | +50% |
| Frequency 한계 | 피로도 경고 임계값 | 3.0 |
| 최소 전환 (LIVE) | LIVE 판정을 위한 최소 7일 전환수 | 3건 |

### 4.4 적응형 CPA 블렌딩 공식

```
effective_CPA = (w × target_CPA) + ((1 - w) × actual_avg_CPA)

w = 슬라이더 값 (0.0 ~ 1.0)
  w = 1.0  → 순수 목표 CPA 기준 (이상적)
  w = 0.0  → 순수 실적 평균 CPA 기준 (현실적)
  w = 0.3  → 기본값 (목표 30% + 실적 70%)

예시:
  target_CPA = 15,000원, actual_avg_CPA = 22,000원, w = 0.3
  effective_CPA = (0.3 × 15,000) + (0.7 × 22,000) = 4,500 + 15,400 = 19,900원
```

### 4.5 분석 윈도우
- **7일 윈도우:** 전체 성과 기준 (LIVE/OFF 1차 판정)
- **3일 윈도우:** 최근 추세 감지 (악화 중인지 확인)
- 추천 데이터 기간: 14일 (7일+3일 비교 + 벤치마크 신뢰도 확보)

### 4.6 업종별 프리셋

| 업종 | 기본 목표 CPA | Frequency 한계 | 특이사항 |
|------|--------------|----------------|---------|
| 패션/의류 | 15,000원 | 3.5 | 충동구매 특성상 빠른 판정 |
| 뷰티/화장품 | 12,000원 | 3.0 | - |
| 식품/건강식품 | 18,000원 | 4.0 | 재구매 고려 |
| 교육/강의 | 30,000원 | 2.5 | 고관여, 긴 구매 사이클 |
| SaaS/앱 | 25,000원 | 3.0 | 무료체험 전환 기준 |
| 인테리어/가구 | 8,000원 (리드) | 2.0 | 리드 전환 기준 |
| 고가 제품 (50만↑) | 50,000원 | 2.5 | 고관여, 낮은 CTR 기준 |

## 5. 데이터 구조 — 캠페인 계층

### 5.1 계층 구조

```
캠페인 (Campaign)
└── 광고세트 (Ad Set / Ad Group)   ← 오디언스, 예산, 일정 결정
    └── 광고 소재 (Ad / Creative)  ← 이미지/영상/카피
```

**왜 중요한가:**
동일 소재라도 어떤 광고세트(오디언스)에 배치됐느냐에 따라 성과가 극적으로 다르다.
소재만 보고 OFF하면 실제로는 오디언스 문제인데 소재를 버리는 오판이 발생한다.

### 5.2 계층별 판정 결과 표시

| 뷰 | 설명 |
|----|------|
| **소재 뷰 (기본)** | 소재별 판정 결과. 동일 소재가 복수 광고세트에 있으면 합산 |
| **광고세트 뷰** | 광고세트별 성과 요약. "이 오디언스 자체가 문제인가?" 진단 |
| **캠페인 뷰** | 캠페인 전체 성과 요약 |

### 5.3 확장 컬럼 매핑

| 우리 내부 필드 | Meta 컬럼 | 네이버 컬럼 | Google 컬럼 |
|--------------|----------|-----------|------------|
| **campaign_name** | 캠페인 이름 / Campaign name | 캠페인명 | Campaign |
| **adset_name** | 광고 세트 이름 / Ad set name | 광고그룹명 | Ad group |
| creative_name | 광고 이름 / Ad name | 소재명 | Ad name |
| spend | 지출 금액 / Amount spent | 비용 | Cost |
| impressions | 노출 / Impressions | 노출수 | Impressions |
| clicks | 링크 클릭 / Link clicks | 클릭수 | Clicks |
| conversions | 결과 / Results | 전환수 | Conversions |
| **conversion_value** | 구매 전환 값 / Purchase value | 전환 매출액 | Conv. value | ← ROAS 계산에 필요
| date | 날짜 / Date | 날짜 | Day |
| ctr | CTR (링크 클릭률) | CTR | CTR |
| cpc | CPC (링크 클릭당 비용) | CPC | Avg. CPC |
| frequency | 빈도 / Frequency | - (N/A) | - (N/A) |
| **conversion_type** | 결과 유형 / Result type | 전환 유형 | Conversion action | ← 전환 종류 구분 |

> **굵게 표시된 필드**가 이번 버전에서 신규 추가됨.

## 6. 데이터 입력

### 6.1 CSV/Excel 업로드
- Meta 광고 관리자 일별 내보내기 파일 그대로 사용
- 한글/영문 컬럼명 자동 매핑
- 필수: 소재명, 노출, 지출, 클릭, 전환(결과), 날짜
- 권장 추가: 캠페인명, 광고세트명, 전환 매출액(ROAS 모드 시 필수)

### 6.2 Meta API 직접 연동
- 환경: 서버 사이드에서 `META_API_KEY` 사용 (서버에서 호출 후 클라이언트에 결과만 전달)
- 토큰 만료: User Access Token은 60일 만료. 장기 운영 시 System User Token 전환 필요
- 호출 레벨: `/{ad-account-id}/ads?fields=name,adset_id,campaign_id,insights,adset{learning_phase_info}`
- 학습 상태: `adset.learning_phase_info.status`로 직접 조회

### 6.3 플랫폼별 API 연동 준비 현황

| 플랫폼 | API 키 상태 | 연동 방식 | 비고 |
|--------|-----------|----------|------|
| Meta | `META_API_KEY` — User Token (60일 만료) | 서버 사이드 호출 | System User Token 전환 권장 |
| 네이버 | `NAVER_AD_API_KEY` + `CUSTOMER_ID` + `SECRET_KEY` — 정식 발급 | HMAC-SHA256 서명 | 사용 가능 |
| Google Ads | `DEVELOPER_TOKEN` (기본 액세스) + OAuth2 Refresh Token | REST API | 기본 액세스로 내부 툴 운영 충분 |
| Kakao | 미확보 | CSV 전용 (Phase 2) | 공식 API 파트너 여부 확인 필요 |

## 9. 사용성 설계 원칙 (주니어 친화)

1. **판정 모드 선택 UI:** 시작 시 CPA/ROAS 모드 선택 → 해당 모드에 맞는 입력값만 표시
2. **계층 드릴다운:** 소재 결과 클릭 → 해당 광고세트의 다른 소재들 성과 비교 가능
3. **학습 중 안내:** "학습 중" 소재에 마우스 오버 시 "Meta 알고리즘이 최적 오디언스를 찾는 중입니다. 지금 판단하면 정확하지 않습니다." 툴팁 표시
4. **가이드 내장:** 각 설정값에 "?" 버튼 → 설명 제공
5. **오류 예방:** 자주 하는 실수 (일별 분류 안 함, Ad Set 레벨 내보내기 등) 사전 안내
6. **프리셋 우선:** 업종 프리셋 선택 → 이후 미세 조정 순서
7. **판정 이유 명시:** 결과만 주지 않고 "왜 이 판정이 나왔는지" 항상 텍스트로 제공
8. **객단가 자동 계산:** 객단가·마진율 입력 시 목표 CPA/ROAS 자동 계산
